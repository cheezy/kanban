# Stride Configuration

**Note:** Examples below use Unix/Linux/macOS bash syntax. Windows users should see https://github.com/cheezy/kanban/blob/main/docs/WINDOWS-SETUP.md for PowerShell equivalents.

Trunk-based, two developers on SEPARATE clones (one agent per clone — never two
agents in one working tree). Each task is built on its own branch and merged into
LOCAL main only after it is approved. Commits accumulate on local main and are
pushed to origin ONLY when a goal completes (after_goal), after rebasing the
whole goal's worth of commits onto the latest origin/main and re-testing.
Standalone tasks (no parent goal) merge to local main but are NOT auto-pushed —
push those manually.

## before_doing

```bash
git diff --quiet && git diff --cached --quiet || { echo "Working tree is dirty — commit or stash before claiming a task"; exit 1; }
git checkout main
git pull --rebase origin main || { git rebase --abort 2>/dev/null; echo "pull --rebase conflicted with local unpushed commits — resolve manually"; exit 1; }
mix deps.get
mix ecto.migrate
git checkout -B "$TASK_IDENTIFIER"
```

## after_doing

```bash
mix test
mix format --check-formatted
mix credo --strict
mix sobelow --config .sobelow-conf
git diff --quiet HEAD -- || (git add -A && git commit -m "Completed task $TASK_IDENTIFIER: $TASK_TITLE")
```

## before_review

```bash
git checkout main
git merge --ff-only "$TASK_IDENTIFIER" || { echo "main moved since branching — ff-only merge of $TASK_IDENTIFIER failed; resolve manually"; exit 1; }
git branch -d "$TASK_IDENTIFIER"
```

## after_goal

```bash
git checkout main
git fetch origin main
git rebase origin/main || { git rebase --abort 2>/dev/null; echo "rebase onto origin/main conflicted — resolve manually, then re-run after_goal"; exit 1; }
mix test
git push origin main
```
